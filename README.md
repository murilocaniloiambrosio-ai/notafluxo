## About the Project

**notafluxo** is a personal fintech exploration project — combining clean software engineering with financial domain logic. The project explores patterns for handling financial data flows, transaction modeling, and document processing.

This showcase demonstrates the architecture and engineering patterns explored during development.

| Layer | Technology |
|-------|-----------|

## Engineering Focus

- **Domain modeling** for financial transactions and document flows
- **Component-driven UI** with clean separation of concerns
- **Secure credential handling** following production security standards
- **Data validation** patterns suited to financial domain rules

## Local Setup

```bash
# Install dependencies
npm install

# Copy the environment template
cp .env.example .env

# Add your own values to .env (no real credentials provided)

# Start the development server
npm run dev
```

> **Note:** A `.env.example` is provided as a configuration template. The actual environment variables are not included.

## Security Practices

This repository follows production fintech security standards:

- **No secrets, credentials, or fiscal certificates** are committed
- `.gitignore` explicitly excludes `.env*`, certificates, and build artifacts
- All financial data shown is **demonstration data only**
- No real user information or transaction data is present
- All sensitive references have been removed from source code and comments
